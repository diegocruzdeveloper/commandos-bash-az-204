Comandos Bash (Azure CLI) e Docker para o Exame AZ-204

Informação Importante:

Este documento apresenta comandos Bash (Azure CLI) e Docker relevantes para cada uma das competências do exame AZ-204. Note que muitos comandos exigem a substituição de valores como <nome-do-recurso>, <seu-grupo-de-recursos>, etc., por nomes reais.

O site foca no passo a passo usando o portal ou trechos de código C#, com alguns comandos Bash explicitamente mencionados na seção de Event Grid. Para outros tópicos, os comandos gerais da CLI foram inferidos para sua conveniência de estudo.

Desenvolver soluções de computação do Azure (25–30%)

Implementar soluções conteinerizadas

Criar e gerenciar imagens de contêiner para soluções

Comando Docker para construir uma imagem:

            
            docker build . -t <sua-imagem-acr>.azurecr.io/myapp:v1

Constrói uma imagem Docker a partir de um Dockerfile no diretório atual e a tagueia para um Registro de Contêiner do Azure (ACR).

Publicar uma imagem no Registro de Contêiner do Azure:

            
            docker push <sua-imagem-acr>.azurecr.io/myapp:v1

Envia a imagem Docker tagueada para o Registro de Contêiner do Azure (ACR).

Executar contêineres usando a Instância de Contêiner do Azure

O ACI permite executar contêineres sem gerenciar VMs. O comando Azure CLI para criar uma instância de contêiner é:

            
            az container create --resource-group <seu-grupo-de-recursos> --name <nome-da-instancia> --image <sua-imagem>

Cria uma instância de contêiner no Azure Container Instances (ACI) a partir de uma imagem Docker.

Criar soluções usando os Aplicativos de Contêiner do Azure

O Azure Container Apps (ACA) é um ambiente serverless gerenciado. A criação de um aplicativo de contêiner usando a CLI envolve:

            
            az containerapp create --name <nome-do-app> --resource-group <seu-grupo-de-recursos> --environment <nome-do-ambiente> --image <sua-imagem>

Cria um novo aplicativo de contêiner no Azure Container Apps.

Implementar Aplicativos Web do Serviço de Aplicativo do Azure

Criar um Aplicativo de API do Serviço de Aplicativo do Azure

Para criar um App Service (incluindo Web Apps e API Apps) via CLI, você usaria:

Cria um plano do App Service:

            
            az appservice plan create --name <nome-do-plano> --resource-group <seu-grupo-de-recursos> --sku F1

Cria um plano do Serviço de Aplicativo, que define os recursos de computação para os aplicativos web.

Cria um Web App:

            
            az webapp create --name <nome-do-webapp> --resource-group <seu-grupo-de-recursos> --plan <nome-do-plano>

Cria um novo aplicativo web dentro de um plano do Serviço de Aplicativo existente.

Configurar e implementar diagnósticos e registros

Para transmitir logs em tempo real usando a CLI do Azure:

            
            az webapp log tail --name appname --resource-group myResourceGroup

Transmite logs em tempo real de um aplicativo web do Serviço de Aplicativo.

Implantar código e soluções em contêineres

Para implantação contínua ou via ZIP, a CLI teria comandos como:

            
            az webapp deploy --resource-group <seu-grupo-de-recursos> --name <nome-do-webapp> --src-path <caminho-para-o-arquivo-zip> --type zip

Implanta o código de um aplicativo web a partir de um arquivo ZIP.

Definir configurações, incluindo o protocolo TLS (TLS), configurações de API e conexões de serviço

Para configurar via CLI:

Definir configurações de aplicativo:

            
            az webapp config appsettings set --resource-group <seu-grupo-de-recursos> --name <nome-do-webapp> --settings <CHAVE>=<VALOR>

Define uma configuração de aplicativo (app setting) para um aplicativo web.

Definir strings de conexão:

            
            az webapp config connection-string set --resource-group <seu-grupo-de-recursos> --name <nome-do-webapp> --settings <NOME_CONEXAO>=<STRING_CONEXAO> --connection-string-type Custom

Define uma string de conexão para um aplicativo web.

Implementar o dimensionamento automático

A CLI do Azure para configurar o autoescalamento envolveria:

            
            az monitor autoscale rule create --resource-group <seu-grupo-de-recursos> --name <nome-da-regra> --resource <ID-do-App-Service-Plan> --condition "MetricName 'CpuPercentage' GreaterThan 70" --scale out 1

Cria uma regra de autoescalamento para um plano do Serviço de Aplicativo, baseada numa métrica (ex: CPU).

Configurar slots de implantação

Na CLI do Azure, você usaria:

            
            az webapp deployment slot create --resource-group <seu-grupo-de-recursos> --name <nome-do-webapp> --slot <nome-do-slot>

Cria um novo slot de implantação para um aplicativo web.

            
            az webapp deployment slot swap --resource-group <seu-grupo-de-recursos> --name <nome-do-webapp> --slot <nome-do-slot> --target-slot production

Troca o conteúdo de um slot de implantação com o slot de produção.

Implementar Funções do Azure

Criar e configurar um aplicativo do Azure Functions

Para criar um Function App via CLI:

            
            az functionapp create --resource-group <seu-grupo-de-recursos> --consumption-plan-location <localizacao> --runtime <linguagem> --functions-version 4 --name <nome-do-functionapp> --storage-account <nome-da-conta-de-armazenamento>

Cria um novo aplicativo de função do Azure (Function App).

Implementar gatilhos função usando operações de dados, temporizadores e webhooks

Os comandos da CLI seriam para criar a função dentro do Function App existente:

            
            az functionapp function create --resource-group <seu-grupo-de-recursos> --name <nome-do-functionapp> --function-name <nome-da-funcao> --template "HttpTrigger" --language "C#"

Cria uma nova função dentro de um Function App existente, usando um modelo (ex: HttpTrigger).

Desenvolver para armazenamento do Azure (15–20%)

Desenvolver soluções que usam o Azure Cosmos DB

Executar operações em contêineres e itens usando o SDK

Para criar a conta Cosmos DB via CLI:

            
            az cosmosdb create --name <nome-da-conta> --resource-group <seu-grupo-de-recursos> --kind GlobalDocumentDB --default-consistency-level Session

Cria uma nova conta do Azure Cosmos DB.

            
            az cosmosdb sql database create --account-name <nome-da-conta> --name <nome-do-banco> --resource-group <seu-grupo-de-recursos>

Cria um novo banco de dados SQL (API Core) dentro de uma conta Cosmos DB.

            
            az cosmosdb sql container create --account-name <nome-da-conta> --database-name <nome-do-banco> --name <nome-do-container> --partition-key-path /<chave-de-particao> --throughput 400 --resource-group <seu-grupo-de-recursos>

Cria um novo contêiner SQL (coleção) dentro de um banco de dados Cosmos DB.

Desenvolver soluções que usam o Armazenamento de Blobs do Azure

Definir e recuperar as propriedades e os metadados

Para criar um contêiner via CLI:

            
            az storage container create --name <nome-do-container> --account-name <nome-da-conta-de-armazenamento> --public-access off

Cria um novo contêiner de blob no Armazenamento de Blobs do Azure.

Implementar políticas de armazenamento e gerenciamento do ciclo de vida de dados

Para gerenciar via CLI (exemplo para aplicar uma política JSON):

            
            az storage account management-policy create --account-name <nome-da-conta> --resource-group <seu-grupo-de-recursos> --policy @policy.json

Cria ou atualiza uma política de gerenciamento do ciclo de vida para uma conta de armazenamento.

Implementar a segurança do Azure (15-20%)

Implementar autenticação e autorização de usuário

Autenticar e autorizar usuários usando a plataforma Microsoft Identity

Na CLI, o registro de um aplicativo para Microsoft Identity Platform:

            
            az ad app create --display-name <nome-do-app> --sign-in-audience AzureADandPersonalMicrosoftAccount

Cria um novo registro de aplicativo no Azure Active Directory para autenticação.

Obtém o ID do cliente:

            
            az ad app show --id <app-id> --query appId --output tsv

Mostra os detalhes de um registro de aplicativo, incluindo o ID do aplicativo (cliente).

Criar e implementar assinaturas de acesso compartilhado (SAS)

Para gerar um SAS via CLI:

SAS para um blob com permissão de leitura:

            
            az storage blob generate-sas --account-name <nome-da-conta> --container-name <nome-do-container> --name <nome-do-blob> --permissions r --expiry <data-de-expiracao-YYYY-MM-DDTHH:mmZ>

Gera um token de Assinatura de Acesso Compartilhado (SAS) para um blob específico, concedendo permissões limitadas.

SAS para um contêiner com permissões de leitura e listagem:

            
            az storage container generate-sas --account-name <nome-da-conta> --name <nome-do-container> --permissions rl --expiry <data-de-expiracao-YYYY-MM-DDTHH:mmZ>

Gera um token de Assinatura de Acesso Compartilhado (SAS) para um contêiner, concedendo permissões limitadas.

Implementar soluções seguras do Azure

Dados seguros de configuração de aplicativo usando a Configuração de Aplicativos ou o Azure Key Vault

Para criar um Key Vault via CLI:

            
            az keyvault create --name <nome-do-keyvault> --resource-group <seu-grupo-de-recursos> --location <localizacao>

Cria um novo Azure Key Vault para armazenar segredos, chaves e certificados.

Para adicionar um segredo:

            
            az keyvault secret set --vault-name <nome-do-keyvault> --name <nome-do-segredo> --value <valor-do-segredo>

Define um novo segredo no Azure Key Vault.

Implementação de identidades gerenciadas para recursos do Azure

Para habilitar uma identidade gerenciada atribuída pelo sistema em um App Service:

            
            az webapp identity assign --resource-group <seu-grupo-de-recursos> --name <nome-do-webapp>

Habilita uma identidade gerenciada atribuída pelo sistema para um aplicativo web do Serviço de Aplicativo.

Criar uma instância do Gerenciamento de API do Azure

Para criar via CLI:

Desenvolver soluções baseadas em evento

Implementar soluções que usam Grade de Evento Azure

Configuração de variáveis:

            
            let rNum=$RANDOM*$RANDOM
myLocation=<myLocation>
myTopicName="az204-egtopic-${rNum}"
mySiteName="az204-egsite-${rNum}"
mySiteURL="https://${mySiteName}.azurewebsites.net"

Define variáveis para uso em comandos do Azure Event Grid.

Criação de grupo de recursos:

            
            az group create --name az204-evgrid-rg --location <myLocation>

Cria um novo grupo de recursos para organizar recursos do Azure.

Registro do provedor de recursos:

            
            az provider register --namespace Microsoft.EventGrid

Registra o provedor de recursos do Event Grid na sua subscrição.

            
            az provider show --namespace Microsoft.EventGrid --query "registrationState"

Verifica o estado de registo do provedor de recursos do Event Grid.

Criação de tópico personalizado:

            
            az eventgrid topic create --name $myTopicName --location $myLocation --resource-group az204-evgrid-rg

Cria um tópico personalizado do Azure Event Grid.

Implantação de web app (Event Grid Viewer):

            
            sitename=<your-site-name>
az deployment group create --resource-group az204-evgrid-rg --template-uri "https://raw.githubusercontent.com/Azure-Samples/azure-event-grid-viewer/master/azuredeploy.json" --parameters siteName=$sitename hostingPlanName=viewerhost

Implanta um aplicativo web (Event Grid Viewer) usando um modelo ARM.

Assinatura de evento:

            
            az eventgrid event-subscription create --name <nome-da-assinatura> --source-resource-id <ID-do-topico> --endpoint <URL-do-endpoint-web-app>/api/updates/

Cria uma subscrição de evento para um tópico do Event Grid, direcionando os eventos para um endpoint.

Disparo de evento (exemplo de payload e curl):

            
            event='[ {"id": "'"$RANDOM"'", "eventType": "recordInserted", "subject": "myapp/vehicles/motorcycles", "eventTime": "' date +%Y-%m-%dT%H:%M:%S%z '", "data":{ "make": "Ducati", "model": "Monster"},"dataVersion": "1.0"} ]'
curl -X POST -H "aeg-sas-key: $key" -d "$event" $endpoint

Dispara um evento personalizado para um tópico do Event Grid usando `curl`.

Limpeza de recursos:

            
            az group delete --name gridResourceGroup

Elimina um grupo de recursos e todos os recursos contidos nele.

Implementar soluções que usam o Hub de Evento Azure

Para criar via CLI:

            
            az eventhubs namespace create --name <nome-do-namespace> --resource-group <seu-grupo-de-recursos> --location <localizacao> --sku Basic

Cria um novo namespace do Azure Event Hubs.

            
            az eventhubs eventhub create --name <nome-do-eventhub> --namespace-name <nome-do-namespace> --resource-group <seu-grupo-de-recursos> --partition-count 2 --message-retention 1

Cria um novo Event Hub dentro de um namespace existente.

Desenvolver soluções baseadas em mensagem

Implementar soluções que usam o Barramento de Serviços Azure

Para criar via CLI:

            
            az servicebus namespace create --resource-group <seu-grupo-de-recursos> --name <nome-do-namespace> --location <localizacao> --sku Standard

Cria um novo namespace do Azure Service Bus.

            
            az servicebus queue create --resource-group <seu-grupo-de-recursos> --namespace-name <nome-do-namespace> --name <nome-da-fila>

Cria uma nova fila dentro de um namespace do Service Bus.

Implementar soluções que usam filas de Armazenamento de Fila Azure

Para obter a string de conexão via CLI:

            
            az storage account show-connection-string --name <nome-da-conta-de-armazenamento> --resource-group <seu-grupo-de-recursos> --query connectionString --output tsv

Obtém a string de conexão de uma conta de armazenamento.

Para criar uma fila via CLI (assumindo que a conta de armazenamento já existe):

            
            az storage queue create --name <nome-da-fila> --account-name <nome-da-conta-de-armazenamento>

Cria uma nova fila no Armazenamento de Filas do Azure.
